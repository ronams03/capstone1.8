## Product Roadmap Phases

This file stores the agreed implementation phases so they stay available in-project.

## Current Status
- Active phase: Phase 2 (started March 3, 2026)
- Phase 1: Completed
- Phase 2: In progress
- Phase 3: Planned

## Phase 1 (Completed, MVP, 3-4 weeks)
1. Manager approval inbox with SLA reminders (`leave`, `overtime`, `payslip disputes`)
2. Staff payslip dispute workflow (submit, track, resolve)
3. Admin exception center (all failed/flagged records in one queue)
4. Audit trail hardening (who changed what, before/after values)

## Phase 2 (Active, 4-6 weeks)
1. Compliance export module (`SSS`, `PhilHealth`, `Pag-IBIG`, `BIR`) with export history
2. Team workload dashboard for managers (overdue tasks, attendance variance, leave calendar)
3. Staff self-service attendance correction requests with manager approval
4. Notification rules engine (email/in-app reminders by SLA and status)

## Phase 3 (4-6 weeks)
1. Backup restore drill page with validation report
2. Role/permission matrix UI with change history and rollback
3. Client portal (documents, project milestones, invoices) if external access is needed

## Phase 2 Priority Order (Current)
1. Compliance export module (`SSS`, `PhilHealth`, `Pag-IBIG`, `BIR`) with export history
2. Team workload dashboard for managers (overdue tasks, attendance variance, leave calendar)
3. Staff self-service attendance correction requests with manager approval
4. Notification rules engine (email/in-app reminders by SLA and status)

## Feature Ownership by Role
- Admin: exception center, compliance exports, restore drill, RBAC matrix
- Manager: approval inbox, team dashboard
- Staff: disputes, attendance correction, leave planning
